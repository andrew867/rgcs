# RGCS v2 — System Architecture

**Author:** Sub-Agent 09. **Date:** 2026-07-14. Companion documents:
`CORE_API_SPEC.md` (library-level contract) and `DESKTOP_ARCHITECTURE.md`
(desktop internals). This document is the system-level view.

## 1. Layering (strict, one-way)

```
manuscript/  tools/make_*.py           (build-time consumers)
        │
rgcs_desktop/                          (PySide6 UI, workspaces, jobs)
        │  imports only public rgcs_core API
rgcs_core/                             (pure computation; no UI, no I/O
        │                               beyond provenance file hashing)
docs/model_registry.yaml               (equation contract RGCS-M.1..M.61)
```

Rules (QA-verified: no leakage in either direction):
- `rgcs_core` never imports Qt, never touches workspaces, never prints.
- `rgcs_desktop` performs no physics: every number displayed comes from a
  core call; viewers carry the core's classification metadata into badges.
- `tools/` and the manuscript consume the same public API as the desktop.

## 3. rgcs_desktop subsystems

- **app/** — entry point (guarded startup auto-open with backup restore,
  QA-D-05), `AppContext` (workspace + job manager + settings), main window
  (13 panels, sidebar, inspector, command palette).
- **workspaces/** — one directory per project; SQLite registry
  (objects/files/jobs/exports) + content-addressed artifacts; atomic
  writes; backup-on-open; integrity check before backup; `WorkspaceError`
  on corruption; `Workspace.restore_latest_backup()`.
- **jobs/** — process-based job manager (spawn), progress/log streaming,
  cancellation; workers for coherence analysis (CSV loader with
  user-facing errors and finiteness gate, QA-D-06/13).
- **services/** — schema validation (strict, versioned, friendly errors
  incl. malformed JSON, QA-D-07), claim gates (coherence claim gate,
  ethics gate), formatting.
- **viewers/, widgets/, plots/** — panels, classification badges,
  uncertainty bands.

## 4. Build/packaging paths

- Library: `pip install .` (pyproject; extras `desktop`, `dev`).
- Desktop binary: PyInstaller spec `tools/packaging/rgcs_desktop.spec`;
  Linux build script `tools/packaging/build_linux.sh` (artifact under
  `release/linux/`); Windows path documented in
  `tools/packaging/build_windows.md`.
- Manuscript: `python3 tools/make_figures.py && python3 tools/make_tables.py
  && latexmk -xelatex manuscript/rgcs_v2.tex`.

## 5. Key architectural decisions

1. **Registry-first equations** — code cannot ship an equation without a
   registry id; provenance decorators make labels machine-checkable.
2. **Complex canonical dynamics** — M.46 integrated in complex form
   (polar form singular at A→0); exponential integrating factor for the
   stiff linear part.
3. **Content-addressed artifacts** — deterministic ids (sha256 of
   canonical JSON) make identical results collide instead of duplicate.
4. **Process-isolated jobs** — analysis crashes become failed jobs with
   preserved tracebacks, never app crashes.
5. **Golden data as contract** — expected values live in
   `experiments/sample_data/golden_coherence/manifest.json`, read by tests
   and regenerated byte-identically by `tools/generate_golden_coherence.py`.
