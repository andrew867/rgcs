# LinkedIn Post

After an intensive development programme, I've released **RGCS v3.0.0**
— an open-source research framework that I believe demonstrates
something useful about how research software should handle uncertain
knowledge.

The domain is resonance in engineered quartz geometries, and the
project's questions come partly from unconventional historical sources.
That constraint forced the interesting engineering: a system where
**epistemic status is a type system**.

What that means in practice:

▪ Five machine-checked claim labels on every function, document, and
manuscript statement — and a build-failing firewall that prevents
"established" results from being computed from unproven inputs
▪ A byte-frozen predecessor version, with CI proving on every commit
that the new mathematics reproduces the old wherever they overlap
▪ Equation-level provenance for every piece of adapted literature math,
with binding "the math crosses, the conclusion doesn't" clauses
▪ 30 pre-registered hypotheses with observables, controls, and failure
conditions — several registered as expected *null* results
▪ Four manuscripts in which no number is hand-typed; three-OS CI;
a complete laboratory validation plan

And the line I'm most deliberate about: **no physical hypothesis is
confirmed**, and every claim-bearing surface of the project says so.
The deliverable is a reproducible framework and an honest falsification
plan — which I'd argue is the correct shape for research software
*before* decisive evidence exists, in any field.

If you work on research software, reproducibility, or scientific QA,
the design-philosophy document is a ten-minute read I'd genuinely value
critique on. MIT licensed.
