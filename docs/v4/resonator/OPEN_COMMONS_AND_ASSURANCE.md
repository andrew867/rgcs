# Hydrogenuine open commons core and commercial assurance boundary
(Agent P02)

A Linux-like open-core structure: a genuinely useful local-first
system stays free to inspect, run, modify, fork, and preserve;
commercial offerings sell **assurance**, not access.

## The commons core (free forever, genuinely useful)

- The full RGCS/RSCS computational stack as released (MIT): solvers,
  registries, capability firewall, Eye engine, resonator platform,
  reference models, all documentation.
- The Hydrogenuine local-first basic memory system: spatial-temporal
  event records, local world-memory blocks, replay/consolidation,
  audit receipts — complete and unthrottled. **No artificial feature
  starvation**: the free core is the system, not a demo of it.
- Every schema, certificate format, and verification tool — a
  certificate anyone can issue must be a certificate anyone can
  verify.
- The claim cards, correction trails, and safety envelopes.

## What commercial offerings may sell (assurance, not scarcity)

| Offering | What it actually is |
|---|---|
| Certification | a signed attestation that a deployment matches the audited configuration — the attestation is the product, the software is free |
| Accountable support | response-time commitments with a human on the hook |
| Integration & specialization | building it into someone's environment |
| Hosted convenience | running it so the customer doesn't have to |
| Long-term maintenance contracts | the promise of future work |

## The boundary rules

1. **Nothing moves from the commons to the paid side.** Features are
   born free or born commercial; a free capability is never withdrawn
   into a paywall.
2. **The paid side may not be required for correctness, safety, or
   verification.** If a paid tool is needed to check a certificate,
   the certificate format has failed.
3. **Forks are legitimate.** The licence and the docs say so without
   resentment; a fork that outgrows the original is the commons
   working.
4. **No open-core bait**: the free tier's limitations, where any
   exist, are listed publicly and are stable.

## Why this shape

The programme's evidence discipline (append-only ledgers, public
correction trails, verifiable certificates) only earns trust if the
verification path is free. Selling assurance on top of a free,
verifiable core aligns the commercial incentive with the scientific
one: the product is credibility, and credibility requires the
commons.

Status: policy document (binding); the licence remains MIT; no code
