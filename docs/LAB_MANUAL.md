## 2. Roles and discipline

Single-operator compatible. Two hats, never worn simultaneously:
**Operator** (runs sessions, blinded where the protocol says so) and
**Analyst** (runs the pipeline on coded data, unblinds only per SAP §7).
A second person, when available, holds the blind codes; otherwise the
software blind (seeded codes from `control_matrix`, sealed by commit
hash before the session) substitutes, and this substitution is recorded
in every affected run manifest.

## 3. Daily procedure

1. Power on, 30 min warm-up; session header record
   (CALIBRATION_GUIDE §2).
2. Session calibrations: DAQ clock lock, channel latencies, coil
   complement, sensor spot checks; **H-29 phase gate**.
3. Pull today's pre-registered run list (generated by
   `control_matrix(sweep, seed)` from the phase's `timing_program`
   record; the seed was committed BEFORE the session).
4. Dummy-load clearance for any new operating point
   (`safe_drive_check`, dummy_load_validated=true only after the
   instrumented dummy run).
5. Execute runs in the randomized order. No mid-session protocol edits:
   a discovered problem aborts the block, it never "adjusts" it.
6. End: data offload + checksum, pipeline ingest + validation pass,
   backup to two media, bench safe state (outputs disabled, laser
   shuttered, PSU limits verified latched).

## 4. Environmental controls

- Temperature: log continuously; physics runs only within 19–25 °C and
  drift ≤ 1 °C/h; specimen ΔT ≥ 5 °C above ambient = hard stop.
- Humidity: log; flag runs outside 30–60 % RH (adsorbate loading is a
  declared H-08-family systematic).
- Acoustic background: SPL floor logged before each block; runs paused
  when floor rises > 10 dB over the commissioning baseline.
- EMI: mains frequency and a no-drive sensor noise floor recorded per
  block (artifact channels); switching supplies inventoried.
- Vibration: isolation platform verified in commissioning (tap
  transfer); re-verified after any bench change.

## 5. Blinding and randomization

- Condition order: seeded shuffle (`randomized_order`), seed committed
  in advance; blind codes attached by `control_matrix(…, seed)`.
- Specimen blinding where physically possible: crystal vs glass vs
  dummy enter identical opaque sleeves for the branches where handling
  doesn't reveal them (optical direction runs, sham-timing runs).
- Sham timing: the H-30 branch scrambles inter-channel phase while
  keeping every waveform's amplitude/spectrum — operator cannot
  distinguish sham from live at the console (the timing box does the
  substitution from the sealed schedule).
- Analyst sees coded labels until the SAP §7 unblinding step; the
  unblinding commit is part of the phase report.

## 6. Safety (operator-facing summary; the envelope is D7-003)

**Electrical:** ≤ 30 V, ≤ 3 A peak, ≤ 5 mJ/pulse — PSU limits latched,
firmware/`safe_drive_check` gates on top; all coil work behind the
overcurrent trip; no exposed conductors above 30 V anywhere on the
bench; dummy-load-first for every new operating point.
**Laser:** class ≤ 3R, ≤ 5 mW, fully enclosed beam path, interlocked
lid, shutter closed by default, no reflective tools inside the
enclosure, laser-safety label on the box. Never bypass the interlock.
**Thermal:** specimen and dummy-load temperatures monitored; 5 °C rise
= stop; power resistors on heatsinks.
**Mechanical:** crystals handled over padded surface; lifting/photo
station separate from the live bench.
**Absolute exclusions (restated):** no human or animal exposure to any
drive; no body-connected electrodes; no therapeutic framing of any run;
no operation above the envelope for any reason, including "just once to
see".
**Interlock chain** (hardware, series): enclosure lid, laser shutter,
overcurrent trip, thermal cutoff → output-enable. Test at every session
start (deliberately open the lid: outputs must drop).

## 7. What a result is (and is not)

A measurement enters the registers only through the pipeline with its
uncertainty, controls, and pre-registered analysis attached. Outcomes
per hypothesis are exactly one of: **CONFIRMED-within-band** (the
pre-registered expected result, at the stated confidence),
**NULL** (the pre-registered null — for H-21/H-23 this is the
*expected* outcome), **FAILED** (the failure condition fired — the
hypothesis row is retired to `NEGATIVE_RESULTS.md`), or
**ARTIFACT** (controls implicate the bench; the run block is
quarantined and a defect row opened). No fifth category exists; in
particular there is no "suggestive" outcome anywhere in this programme.
